A constraint satisfaction problem (CSP) is a more generalized version of k-SAT, so we still have n variables
But rather than booleans, these variables can take any value from any arbitrary domain of our choosing

For example these numbers could be integers, or be from a set {Sarah, Manny}.

Then we define some constraints that these variables need to satisfy.

SPARSIFICATION LEMMA
--------------------------------------------------
Scene:
	(x1 or not x3 or x4) and (not x2 or x1 or x3) 
	->
	(x1) or ((not x3 or x4) and (not x2 or x3)) Highlight both CNF in blue

Voiceover:
We start by using the sparsification lemma. This tells us that any k-SAT formula can be rewritten as a disjunction of at most 2^epsilon n sparse k-SAT formulas. Sparse meaning that each variable only appears a constant amount of times c_k,e. Here epsilon is a small constant that represents a tiny exponential overhead. This transformation requires poly(n)2^epsilon n time.

For example, our CNF formula from before can be rewritten as a disjunction (orring) of two CNFs like shown. Where each blue part is a sparse CNF.

Voiceover:
Each of our CSPs will correspond to solving a sparse CNF term. Note that since we perform a disjunction over all instances, if the k-SAT formula is satisfiable, at least one instance of CSP will be satisfiable. And if the formula is not satisfiable, then none of the instances of CSP will be satisfiable.
As a reminder our CSP needs variables that have a domain and constraints we need to satisfy. Our variables for CSP will represents blocks of variables of some parameter size a >= 1 in our k-SAT. For demonstration purposes, let's say there are 10 variables in our k-SAT formula, and a = 4. Then our CSP variables, y1, y2, y3 will represent these blocks of variables. For example, y2 represents the block x5, x6, x7, x8

Since each x variable is a boolean which can be represented with a bit, we can encode every super variable y as a binary string of length a. 

For example y2 = 0011, corresponds to x5 and x6 being false and x7 and x8 being true

To recap our size,
psi_i has n hat variables = ceil(n/a), that are over a domain 2^a

For the constraints of our CSP, we'll do a similar partitioning into super-constraints. It's important to note that rather than clauses over singular variables, these clauses now represent the corresponding bit in the super variable being set. 

First we partition our clauses into blocks of size a, like we did for the variables. But then we combine these constraints into groups of size gamma (which we will go over into more detail shortly).

Voiceover:
So what was the point in reframing our k-SAT into an exponential amount of structured CSP problems that represent sparse CNF formulas? The benefit is that it lets us analyze the problem's complexity more cleanly using known tools and results.

Since we group variables into blocks of size a, each variable is only part of a * C_k, e clauses. We label this gamma, which is what we use to determine the size of our super constraints.
