Each CSP variable no longer represents a single Boolean variable.

every variable participates in at most  ĉₖ,ε · a constraints,

DESCRIBING THE INTEGER ITEMS 
--------------------------------------------------
Scene:
Integer Types

Variable Items:
    z(x, alpha) <- notation
    z(x_1, 4) -> x_1 = 4
    x_1 = 4  x_1 = 5   <- x_1 type
    x_2 = 2  x_2 = 3   <- x_2 type
Constraint Items:
    z(c, alpha1, alpha2, ...) <- notation
    z(c_1, 1, 2)
    c_1 = (x_1 != x_2)     <- c_1 type
    c_2 = (x_1 = 5 -> x_2 = 3)  <- c_2 type

Voiceover:
We can classify the input integers we'll create for Subset Sum as Variable Items 
and Constraint Items.

Variable Items represent assigning a variable to a certain value. 
For every possible assignment of a variable, there is an associated Variable Item.

Distinct variables have their own type. For example, x_1 assignment integers are a 
different type from x_2 assignment integers.

Constraint Items represent an assignment of variables that satisfy a constraint. 
There is a Constraint Item for every satisfiable assignment of variables of a constraint. 

Similar to Variable Items, distinct constraints have their own type.

Voiceover:

Each variable block represents assignment consistency for that variable. 

A solution subset needs to sum to some value for this block that we'll call Q for now. 
So, T has Q in each instance of a variable block. We'll revisit the amount of bits in these blocks.

Let's set for each variable item, the block corresponding to that variable to be 
Q - d(x)*alpha where d(x) is the number of constraints that contain the variable 
and alpha is the value that the variable was assigned.

If it's a constraint item that uses the variable, then we'll set the block to be 
the value that it sets the variable as alpha_i. If the constraint doesn't use the variable,
then the block will be 0.

CONSISTENCY 
--------------------------------------------------
Scene:

Q - d(x)*alpha + sum from i=1 to d(x) alpha_i

k-Average-Free Set S

Definition:
No k numbers in S have an average in S  
unless all chosen numbers are identical.

lamda = ĉₖ,ε · a


Voiceover:
The sum of a set of chosen items would be Q - d(x)*alpha + sum from i=1 to d(x) alpha_i.
For the value to match up with the target block, d(x)*alpha would need to equal 
sum from i=1 to d(x) alpha_i. However, this equality could be achieved with values alpha_i 
that are not equal to the variable's alpha.

This is why the paper introduces k-average-free sets. 
A k-average-free set S guarantees that the average of a selection of k numbers 
from S is not present in S. And that the sum of k values in S can only equal 
a multiple of another value in S if all of the values chosen from S are equal.



We can map our assignment values to an average-free set. 
Let lambda be ĉₖ,ε · a and be the max amount of constraints that a given variable could be in.

Let S be a lamda-average-free set.

Let f be a mapping function from alpha_i to an element in S.

T VARIABLE BLOCK value
--------------------------------------------------
Scene:
lamda-average-free set 
Size = 2^a
B = lambda^{c/e}{2^a}^{1 + e}

Subtraction term:
Q - d(x)*f(alpha)

Q = d(x)*f(alpha)
lambda = max possible d(x)

Q = lambdaB

Voiceover:
Lemma 3.2 in the paper states that a lamda-average-free set S of size 2^a,
 values in f(alpha) will be bounded between [0, lambda^{c/e}{2^a}^{1 + e}]
in poly(n) time.

Let's have B be the max value in the set S.
We need the variable block at any point in addition to be non-negative; however variable
items involves subtraction terms. We need Q to be equal to the max value possible of 
d(x)*f(alpha). 

We know that the maximum value possible for d(x) is lambda and the max value possible 
for f(alpha) is B.

Therefore the max possible value of d(x)*f(alpha) is lambda B.
So each variable block in T will be lambdaB.

This avoids negative values.


VARIABLE BLOCK size
--------------------------------------------------
Scene:

Addition term:
sum from i=1 to d(x) f(alpha_i)

2lambdaB

Voiceover:
Now, what should each variable block's size be.

Well with the addition terms throughout constraint items, our max value possible within a 
variable block would be ... (let the equations play through) 2lambdaB.

To represent the range [0,2lambdaB], each variable block needs to be of size ceil(log(2lambdaB + 1))

With these variable blocks, we'll be able to check consistency of an assignment 
for each variable without worrying about overflowing into other blocks.
