FINE GRAINED COMPLEXITY
--------------------------------------------------
Scene:
Fine Grained Complexity:
	Assumption: Problem A requires at least X time to solve
	Show: Problem A can be solved with problem B
	Conclusions: Then problem B must take at least X time

	Reducing problem A to problem B: Reduction

Voiceover: Fine grained complexity is a relatively recent field that has to do with how much we can actually optimize algorithms. The idea is to start with a classic, well-studied problem A for which we believe the best possible running time is already known. Then, we take another problem B (the one we are interested in) and show how to solve A using B. This is called a reduction from A to B. If we can show this reduction of A to B, then this implies B is at least as hard as A. More specifically, that B will take at least X time to solve. Otherwise a faster algorithm for B would give us a faster algorithm for A. Although fine-grained reductions can involve technical details about precise time bounds, the core idea is to transfer hardness, lower bounds on running time, between A and B
